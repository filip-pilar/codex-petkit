# Standard-state distinction planning

Write `qa/standard-motion-plan.md` before generating standard rows. For every state record:

- semantic intent and emotional valence;
- primary normal-size silhouette signature;
- face/eye cue;
- rhythm and hold pattern;
- ordered frame beats matching the contract count;
- motion amplitude (`micro`, `small`, `medium`, or `large`);
- planted baseline/scale rule;
- `must not resemble` states and the concrete visual difference.

Actual character scale stays consistent. “Amplitude” means how far posture, limbs, head, and center of mass move inside the cell—not resizing the character.

## Required anti-confusion matrix

| Pair | First state must read as | Second state must read as |
|---|---|---|
| idle / waiting | closed, calm, self-contained micro-motion | open, user-facing request with an expectant hold |
| waiting / running | paused for external input | internally driven, quick alternating active work |
| running / review | repeated busy manipulation or scanning | slow asymmetrical scrutiny with deliberate holds |
| review / failed | upright or forward analytical evaluation | lowered, inward emotional collapse and recovery |
| failed / idle | visibly sad, worried, or deflated | emotionally neutral and physically settled |

Give each side a different primary silhouette and a different timing signature. Face-only differences do not pass because they may disappear at 192×208. If the character lacks hands or a separate head, translate the same semantics into body deformation, eye mechanics, appendage angle, center-of-mass movement, or one controlled prop authorized by the capability audit. Do not force several abstract states through tiny eye changes alone.

## Final review

Review each GIF individually, then compare every required pair side by side at normal pet size and approximate Codex UI size. A pair fails when a viewer could swap the app-state labels without noticing a mismatch. Repair the complete failing standard row and preserve the semantic signature. A simple controlled prop may solve a demonstrated capability gap; decorative text, punctuation, UI labels, symbols, or detached effects are not semantic substitutes.
