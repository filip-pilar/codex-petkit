# Edit semantic design

An edit preserves or intentionally improves the row's observable semantic signature. Before prompting, record the visible meaning, primary full-size and UI-size silhouette, body rhythm, gaze target, frame beats, anti-confusion constraint, and the single capability or approved stable prop carrying the action.

Describe the observable action before the internal state name. If the requested edit cannot be recognized without its label at approximate Codex UI size, reject the concept or redesign the complete row. Do not use tiny eye changes, decorative text, detached symbols, or an explanation of intent as a substitute for a readable action.

The edit must preserve canonical identity, model scale, material, lighting, and support/contact logic. A simple state-local prop is allowed only when the capability audit authorizes it as the decisive cue and the prop remains stable, legible, and physically attached or consistently placed across the row.
