# Edit capability audit

Inventory the current character's body, appendages, eye/face cues, interaction surfaces, asymmetry, and silhouette before a generative edit. Map the requested state to at least one strong visible capability. If the edit depends on a capability the character does not have, redesign the complete row or authorize one simple stable prop before prompting.

The audit is a gate: a technically clean edit that leaves waiting, active work, review, or failure indistinguishable at thumbnail size does not pass. Do not force several abstract states through generic head sway or eye-only motion.
